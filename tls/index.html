SSL/TLS

SSL/TLS состоит из:

  1. Шагов по установлению безопасного соединения (handshake)
  2. Набора алгоритмов дла осуществления безопасного соединения (Ciphersuite)
  3. Цифровых SSL сертификатов для аутентификации сервера

Ciphersuite

Ciphersuite

  1. Шифрование – сокрытие информации, передаваемой от одного компьютера к другому;
  2. Аутентификация – проверка авторства передаваемой информации.

Шифрование

Шифрование

  1. Симметричное
  2. Асимметричное

Симметричное шифрование

Асимметричное шифрование

Шифрование

  1. Ключ для симметричного шифрования - 128 бит
  2. Ключ для асимметричного шифрования - 2048 бит

Шифрование

  1. Симметричное
  2. Асимметричное
  3. Гибридное

Шифрование

  1. Асимметричное - для обмена симметричным ключом
  2. Симметричное - для шифрования сообщений

Протокол Диффи — Хеллмана

Perfect forward secrecy (PFS)

Протокол Диффи — Хеллмана с:

  1. статичным ключом
  2. эфимерным ключом

Алгоритмы:

  1. Асимметричное шифрование: RSA, ECDHE
  2. Симметричное шифрование: AES, CHACHA20

Сертификат

- это файл, использующий электронно-цифровую подпись и связывающий открытый (публичный) ключ компьютера с его принадлежностью.

Certificate Authority (CA)

Задачи CA

  1. Выпуск сертификатов
  2. Проверка подлиности домена:
    • является ли домен реально существующим
    • имеет ли программист доступ к домену, сертификат для которого он пытается получить
  3. Электронная подпись сертификата
